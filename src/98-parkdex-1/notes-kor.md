## 우로보로스

**1. 우로보로스가 뭔지**
- AI 코딩 에이전트(Claude Code, Codex CLI 등)와 사용자 사이에 위치하는 **명세 우선(Specification-first) 워크플로우 엔진**이다.
- 막연한 아이디어를 → 소크라테스식 인터뷰 → 불변 명세(Seed) → 코드 실행 → 자동 검증의 루프로 처리한다.
- 루프는 단순 반복이 아니라 **진화(evolve)**하며, 온톨로지 유사도 >= 0.95에 도달하면 수렴하고 멈춘다.

---

**2. 왜 써야 하는지**
- AI 코딩 실패의 원인은 AI 능력이 아니라 **사람의 불명확한 입력**인데, 우로보로스는 코드를 짜기 전에 그 모호함을 수치(Ambiguity Score <= 0.2)로 강제 해소한다.
- 3단계 자동 검증 게이트(Mechanical → Semantic → Multi-Model Consensus)가 "대충 보기엔 괜찮음" 수준의 수동 QA를 대체한다.
- 명세가 코드보다 먼저 확정되므로, **잘못된 가정이 PR 리뷰 단계가 아니라 인터뷰 단계에서 드러나** 재작업 비용이 줄어든다.

---

**3. 사용하지 않을 경우 불리한 점**
- 명세 없이 AI에게 바로 프롬프트를 던지면 AI가 의도를 추측해 빌드하고, 이미 코드가 완성된 뒤에야 방향이 틀렸음을 발견하게 된다.
- 아키텍처가 중간에 표류(drift)하기 쉬워, 뒤늦게 드러난 가정 오류 때문에 대규모 재작업이 반복된다.
- 검증 기준이 주관적인 "보기엔 됐다"에 머물러, 실제로 올바른 것을 만들었는지 수치적으로 확인할 방법이 없다.

## Anthropic API 의 내부 프로세싱
Anthropic 은 사용자의 요청을 받으면 아래의 프로세스를 밟음

1. 토큰화 : 요청받은 텍스트를 쪼갬
2. 임베딩 : 토큰을 숫치화 시킴
3. 맥락화 : 주변 단어를 기반으로 토큰의 의미를 파악
4. 생성 : 다음 토큰을 예측하여 응답을 생성


## Anthropic API 응답 형태
1. 메시지 : 생성된 응답 텍스트
2. 사용량 : 입력 및 출력 토큰 수
3. 종료이유 : 생성 과정이 종료된 이유

## 프롬프트 파이프라인
프롬프트를 평가 파이프라인을 통해 실행하면 더 광범위한 테스트 케이스에 걸쳐 성능에 대한 객관적인 지표를 얻을 수 있습니다. 
이러한 데이터 기반 접근 방식을 통해 다음과 같은 이점을 누릴 수 있습니다.

- 약점을 파악하여 생산 문제로 발전하기 전에 해결하십시오.
- 서로 다른 프롬프트 버전을 객관적으로 비교하세요
- 측정 가능한 개선 사항을 기반으로 자신감 있게 반복 작업을 진행하세요.
- 더욱 안정적인 AI 애플리케이션을 구축하세요

이 접근 방식은 초기 투자 비용과 테스트 인프라 구축에 더 많은 시간이 소요되지만, 
최종 애플리케이션의 안정성과 견고성을 크게 향상시켜 줍니다. 
목표는 사용자가 문제를 경험한 후에 해결하는 것이 아니라 개발 단계에서 문제를 발견하는 것입니다.

## 프롬프트 작성론 - 직접적
"직접적"이라는 것은 요청을 어떻게 구성하는지에 초점을 맞춥니다.

질문이 아닌 지침을 사용하세요.
"쓰다," "만들다," 또는 "생성하다"와 같은 직접적인 동사로 시작하세요.
"재생에너지에 대해 읽었는데 지열에너지가 꽤 괜찮아 보이네요. 어떤 나라들이 지열에너지를 사용하고 있나요?"라고 묻는 대신, "지열에너지를 사용하는 나라 세 곳을 알려주세요. 각 나라의 발전량 통계도 함께 제시해 주세요."라고 질문해 보세요.

## 프롬프트 작성론 - xml 태그 활용
공식 XML 태그를 사용할 필요는 없습니다. 콘텐츠에 맞는 설명적인 이름을 만드세요.

<sales_records>~보다 낫다<data>
<athlete_information>사용자 정보를 명확하게 식별합니다.
<my_code>그리고 <docs>서로 다른 유형의 콘텐츠를 분리합니다.
태그 이름이 구체적이고 설명적일수록 Claude는 각 섹션의 목적을 더 잘 이해할 수 있습니다.


XML 태그는 다음과 같은 경우에 가장 유용합니다.

많은 양의 맥락이나 데이터를 포함하는 것
코드, 문서, 데이터 등 다양한 유형의 콘텐츠를 혼합합니다.
콘텐츠 경계를 매우 명확하게 설정해야 합니다.
여러 변수를 보간하는 복잡한 프롬프트 작업
짧은 콘텐츠의 경우에도 XML 태그는 구분 기호 역할을 하여 클로드에게 프롬프트 구조를 더욱 명확하게 보여주는 데 도움이 될 수 있습니다.


## 프롬프트 작성론 - 멀티샷, 원샷
예외적인 상황을 처리하기 위한 예제 추가
이 문제를 해결하려면 클로드에게 까다로운 경우를 처리하는 방법을 보여주는 예시를 추가하면 됩니다.

예시는 특히 다음과 같은 경우에 유용합니다.

- 예외적인 경우 또는 특수한 시나리오 포착
- 복잡한 출력 형식(예: 특정 JSON 구조) 정의
- 원하는 스타일이나 톤을 정확하게 보여줍니다.
- 모호한 입력값을 처리하는 방법을 보여줍니다.


## Fine grained 도구
```
run_conversation(
    messages, 
    tools=[save_article_schema], 
    fine_grained=True 
)
```
fine_grained=True 를 하게 되면, 아래의 상황에 대응이 가능하다

- 도구 인자 생성 과정에서 사용자의 실시간 진행 상황을 보여줘야 합니다.
- 부분 도구 결과 처리를 가능한 한 빨리 시작하고 싶을 것입니다.
- 버퍼링 지연은 사용자 경험에 부정적인 영향을 미칩니다.
- 당신은 견고한 JSON 오류 처리를 구현하는 데 능숙합니다.


## 텍스트 편집기 도구를 사용하는 이유 (이미 있는데)
최신 코드 편집기에는 이미 AI 도우미 기능이 내장되어 있는데 왜 이런 도구가 필요한지 궁금할 수 있습니다. 텍스트 편집기 도구는 다음과 같은 상황에서 유용합니다.

- 당신은 파일을 프로그램 방식으로 편집해야 하는 애플리케이션을 개발하고 있습니다.
- 여러분은 모든 기능을 갖춘 코드 편집기를 사용할 수 없는 환경에서 작업하고 있습니다.
- Claude 기반 애플리케이션에 파일 편집 기능을 직접 통합하고 싶으신가요?

기본적으로 텍스트 편집기 도구를 사용하면 고급 AI 기반 코드 편집기의 많은 기능을 자체 애플리케이션에서 구현할 수 있으며, Claude가 파일 시스템과 상호 작용하는 방식을 세밀하게 제어할 수 있습니다.

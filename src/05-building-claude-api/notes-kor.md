
## 모델의 종류

2. sonnet : 균형이 잘 잡힘, 가장 많이 사용됨

## Anthropic API 접근 흐름
> 내 앱 -요청-> 내 서버 -요청-> Anthropic API -응답-> 내 서버 -응답-> 내 앱

1. 서버가 필요한 이유(다이렉트로 붙히면 안되나?) : 보안 이슈


## Anthropic API 의 내부 프로세싱
Anthropic 은 사용자의 요청을 받으면 아래의 프로세스를 밟음

1. 토큰화 : 요청받은 텍스트를 쪼갬
2. 임베딩 : 토큰을 숫치화 시킴
3. 맥락화 : 주변 단어를 기반으로 토큰의 의미를 파악
4. 생성 : 다음 토큰을 예측하여 응답을 생성


## Anthropic API 응답 형태
1. 메시지 : 생성된 응답 텍스트
2. 사용량 : 입력 및 출력 토큰 수
3. 종료이유 : 생성 과정이 종료된 이유

## Temperature 조절
1. 저온(0.0 - 0.3)
  - 사실에 근거한 답변
  - 코딩 지원
  - 데이터 추출
  - 콘텐츠 검열
2. 중간 온도 (0.4 - 0.7)
  - 요약
  - 교육 콘텐츠
  - 문제 해결
  - 제약 조건이 있는 창작 글쓰기
3. 고온(0.8 - 1.0)
  - 브레인스토밍
  - 창작 글쓰기
  - 마케팅 콘텐츠
  - 농담 생성
